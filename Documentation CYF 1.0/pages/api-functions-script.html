Script オブジェクト (及び Encounter 変数) [varies] Scriptオブジェクトは少し特殊で、エンジンにロードされた他のスクリプトを参照するためにあります。

エンカウンタースクリプトでは、戦闘開始時にenemies配列がScriptオブジェクト配列になり、 DEFENDING状態になるとWave配列がScriptオブジェクト配列になります。

Encounter変数もScriptオブジェクトであり、エンカウンタースクリプト以外のどこからでもアクセスできます。 エンカウンタースクリプトのファイル名を使用しないでください。エンカウンタースクリプト用のScriptオブジェクトは常にEncounterです。大文字小文字を区別します。

  • script.GetVar( variable_name) - スクリプト内のvariable_nameと言う名前の変数を取得します。

  • script[ variable_name] - 上記と同様

  • script.SetVar( variable_name, value) - スクリプト内のvariable_nameと言う名前の変数を設定します。

  • script[ variable_name] = value - 上記と同様

  • script.Call( function_name) - スクリプト内のfunction_nameと言う名前の関数を呼び出します。

  • script.Call( function_name, argument) - スクリプト内のfunction_nameと言う名前の関数を １つの引数を渡して呼び出します。

  • script.Call( function_name, any types arguments) - arguments配列内の要素を引数として渡して、 スクリプト内のfunction_nameと言う名前の関数を呼び出します。

    例えば次のように、エンカウンタースクリプト内にあるTakeTwoArgumentsと言う関数を２つの引数を渡して呼び出すことができます: 
                    Encounter.Call("TakeTwoArguments", {"argument 1!", "argument 2!"})
    与えられた２つの引数は任意の型にできます！

  • script.scriptname - this is equal to the file name of the script in question.

Examples

Here are some quick examples:

                -- これはエンカウンタースクリプトで、最初の敵を非アクティブ化しようと思います。
                -- そのためには、モンスタースクリプトで使用できる"SetActive"(「その他の関数」を参照してください。)を使用します。
                enemies[1].Call("SetActive", false)
                

                -- これはエンカウンタースクリプトにある"wavetimer"変数を他のスクリプトから設定する簡単な方法です。
                -- モンスタースクリプトやウェーブスクリプト内で実行できます。
                Encounter.SetVar("wavetimer", 10)
                -- もしくは
                Encounter["wavetimer"] = 10
                

                -- モンスタースクリプトで、エンカウンタースクリプトのスプライトをちりにしたいとしましょう。
                -- それには２つの方法があります。
                local spriteToDust = Encounter.GetVar("torso")
                spriteToDust.Dust(true, true) -- 「スプライト & アニメーション」を参照してください。
                -- もしくは:
                Encounter["torso"].Dust(true, true)
