テキストコマンド



テキストコマンドには２つの種類があります。まず、テキストの色や効果など即座に実行されるもの。 次に、一定時間待機や声などインライン(テキストの中で)実行されるものがあります。 今のところ、テキストをスキップ(Xキーなど)すると、インラインのコマンドもスキップされることに気を付けてください。 ただし、それはplayerskipdocommandで制御できて、スキップされても実行することができます :P

改行: 改行には、２つの種類があります。アスタリスク(*)が自動的に表示されるUIメッセージでは、 \nを使用してアスタリスク(*)付きで改行できます。 改行でアスタリスク(*)が必要ない場合は\rを使用します。

これは、アスタリスク(*)がつかない、モンスターのダイアログ(会話)などでは異なり、常に\nを使用して下さい。 Or use autolinebreak! See Special Variables.

即時(発動 実行)コマンド



[color:rrggbb] このコマンドの後の、全てのテキストの色を指定します。色は16進数で指定します。ダイアログごとに色はリセットされます。 [color:ff0000]は赤色、[color:555555]は濃い灰色になります。
例は以下の通り:

[color:ff0000] Determination (決意)
[color:003cff] Integrity (誠実)
[color:00c000] Kindness (親切)
[color:ffff00] Justice (正義)
[color:d535d9] Perseverance (不屈)
[color:fca600] Bravery (勇気)
[color:42fcff] Patience (忍耐)

色を使用する場合は、上のリストの色を使用することをお勧めします。 このコマンドで任意の色を指定することはできますが、実際のUndertaleはテキストに、これらの色程度しか使用していません。

デフォルトで、UIテキストは無地の白色で、敵のダイアログ(会話)のテキストは無地の黒色です。それぞれ[color:ffffff] と[color:000000]です。


[alpha:aa] テキストのアルファ(透明度)を指定します。

入力する数値は、"00"から"ff"の間の16進数である必要があります。


[starcolor:rrggbb] colorと同じ使用方法で、アスタリスク(*)のあるダイアログで、アスタリスクの色を変更します。 This is a dirty workaround, but now you don't need it anymore, 非推奨になりました。.

[color]コマンドがアスタリスク(*)の前に実行されるようになったため、colorコマンドで色を変えられます。
うまくいった覚えがない... by 翻訳者


[effect:x] or [effect:x,intensity] コマンドの位置に関係なく、メッセージ全体のエフェクトを指定します。 以下のエフェクトを使用できます:

  • none: エフェクト無し

  • rotate: テキストを回転させます。モンスターのダイアログ(会話)にデフォルトで指定されています。 "intensity"は、文字を回転させる距離をピクセル単位で指定します。デフォルトでは1.5です。

  • shake: テキストを揺らします。フラウィが時々使うやつです。 "Intensity"は文字がどれだけ揺れるかをピクセル単位で指定します。デフォルトでは1.0です。

  • twitch: 文字が時々けいれん(揺れる)します。バトルUIにデフォルトで指定されています。 "Intensity"は文字がどこまで揺れるかを指定します。デフォルトでは2.0です。

Note that the twitch effect should, at a later point, let you set shake frequency. Unfortunately you can't do this yet.


[lettereffect:x] or [lettereffect:x,intensity] これは[effect:x] / [effect:x,intensity]と同じですが、インラインで実行できます。 つまり、１行に複数の異なるエフェクトを混在できます。

他の全てのインラインコマンドと同様に、 playerskipdocommandをtrueに指定していないと、 プレイヤーがテキストをスキップした時に実行されないことに注意してください。


[font:x] ダイアログのフォントを指定します。フォントには色や声のデータも含まれていて、声なども自動的に変更されます。 [font]コマンドで色や声も変更されるため、 テキストに独自の声/色を付けたい場合は、フォントの設定後に変えてください。 指定できるフォントの例:

  • uidialog: UIのデフォルトで使用されているピクセルフォント。(英語版)

  • monster: ゲーム内のほぼ全てのモンスターが使用しているフォント。(英語版)

  • sans: sans. 小文字を使用します。大文字も使用できるけど...サンズだぜ...？(英語版)

  • papyrus: PAPYRUS! 大文字だけしか使えません。小文字は対応していないよっ！(英語版)

  • wingdings: Wingdings. ⚐⚐💣⚐☪✋ 🕈⚐ ❄💧🕆😐✌🕆 ❄⚐ ✋✋

  • uibattlesmall: プレイヤー名や、HP、LVに使われているフォント。

  • などなど...独自のフォントを作成することもできます！

使用できる全てのフォントは、Default/Sprites/UI/Fontsフォルダ内の.xmlファイルの名前全てです。


日本語のフォントはJPDefault/Sprites/UI/Fontsフォルダ内の.xmlファイルの名前全てです。以下が現時点で使用できるフォント:

  • jp: UIのデフォルトで使用されているピクセルフォント。(日本語版)
  • jp2: ゲーム内のほぼ全てのモンスターが使用しているフォント...ではないっ！！！

[charspacing:number] and [charspacing:default] 同じ行にある文字同士の間隔をピクセル単位で指定します。調整では...?[要検証]
負の数で、間隔が狭くなり、正の数で間隔が広くなります。

デフォルトでは3です。
フォントファイルは<charspacing>タグでこれを指定できます。

[charspacing:default]と入力することで、フォントデータで指定された値(または3)にリセットします。


[linespacing:number] フォントの行間を指定します。The number you enter here gets added on to the active font's regular line spacing.[要検証]
負の数で、間隔が狭くなり、正の数で間隔が広くなります。

By default, the amount of pixels between lines is 1.5 * the height of the font's "space" character.
If the font has a <linespacing> tag, its number value will be the amount of pixels between lines instead.

Enter [linespacing:0]と入力することで、値をリセットします。

インラインコマンド



[noskip] Xキーなどでテキストをスキップできなくします。

テキストの任意の位置から実行できますが、テキストの最初に配置することで、 テキスト全体がスキップできないようになります。


[noskip:off] [noskip]の実行を取り消せます。


[instant] 即座にテキスト全体を表示します。

* 注: このコマンドはテキストの先頭に置くことで、インラインコマンドとして機能します...!?
This command works as an inline command when placed at the beginning of a line of text.[原文]


[instant:allowcommand] The same as [instant], but here other text commands are called, except [w], [letters] and commands with the "skipover" tag.

* Note: This command works as an inline command when placed at the beginning of a line of text.


[instant:stop] [instant]コマンドまたは[instant:allowcommand]コマンドを取り消します。 and resumes typing.


[instant:stopall] [instant]コマンドまたは[instant:allowcommand]コマンド、 さらにはプレイヤーのスキップさえも取り消します。


[w:x] 一時待機コマンド。代入した数値に合わせてテキストを一時停止します。

技術的情報: これにより、ゲーム内x * 4フレーム間テキストが一時停止します。
[w:1]を呼び出すと、ゲーム内の 4フレーム間テキストがフリーズします。
同様に、[w:4]は 16フレーム、[w:10]は 40フレームです。


[waitall:x] このコマンドの後の全てのテキストが、通常よりx倍の時間をかけて表示されます。 ([w:x]とは異なります。)

[waitall:1]で通常の速度で表示されるようになります。

[speed:x]より強い([speed:x]を指定していても、それを無視して、これで指定された速度になる。)


[voice:filename] 声(文字が表示されるたびになる音)をYOUR MOD/Sounds/Voicesにあるサウンドに設定します。 このコマンドの後の全てのテキストに適用されます。ダイアログごとにリセットされます。 このサウンドは.wavファイルである必要があり、[voice]に指定する際は、ファイルの拡張子を含めないでください。 例えば、指定するサウンドがYOUR MOD/Sounds/Voices/mettaton.wavの場合は、 [voice:mettaton]と指定します。
[voice:default]でデフォルトの音声(ビープ音(?))にリセットされます。If you have a voice sound named 'default', it will be ignored.[翻訳力不足]


[novoice] このコマンドの後の全てのテキストは、声を鳴らしません。ダイアログごとにリセットされます。 じごくで もえたい 時に便利です。
Useful for when you should be burning in hell.[原文]
[next] 自動的に次のダイアログにスキップします。テキストボックストリックにも使えます。Here's an example 以下は、フラウィの「なかよしカプセル」(たｍ)を２回避けた後のテキストを再現する例です。(注: [w]コマンドの長さは合ってるかわかりません。) 

first line:  "[noskip][voice:flowey][effect:none]ふざけてんの？[w:20]\nバカなの？[w:20]\nさっさと　あたれよ…\nたｍ[w:30][next]"
second line: "[instant][effect:none]ふざけてんの？\nバカなの？\nさっさと　あたれよ…\nなかよしカプセルに"


[finished] Sets this dialogue box as "finished". To go through the next dialogue box, you'll have to wait until the other monsters' dialogue boxes are also finished.


[nextthisnow] Skips to the next dialogue of all monsters automatically, without concern for the other monsters' dialogue's state.


[waitfor:key] Waits for the user to press the given key to continue the message. Check the Key List to see the available keys.


[func:x], [func:x,argument] or [func:x,{argument1, argument2...}] めちゃやばさいきょーのコマンド。If the previous text commands were established official characters, [func] is some kid's deviantArt original character that never dies and has all the superpowers.
[翻訳力不足][ユーモア不足][読解力不足]

真面目な話ですが、[func]コマンドを使用して、スクリプトからテキストに、任意の関数を実行できます。 以下は例です。

your dialogue: "hoi hoi this is dog [func:dog] and now the music changed"

function dog()
    Audio.LoadFile("dog_music")
    --Audioファイルにあるdog_music.ogg (or .wav)を再生します。このようなビルドイン(組み込み)関数に関してはAPIセクションの 関数 & オブジェクトを参照してください。
    --任意のコードをもっと追加できます。
end
your dialogue: "dog with arguments!! [func:newmusic,temietheme] so intense!"

function newmusic(yourargumentname)
    Audio.LoadFile(yourargumentname) --この例では'temietheme.ogg (or .wav)'をロードします...
    --そして再生します！未来は来たぜ！引数を使用することで、関数の用途を広げることができます。
end


注意: 関数はソーススクリプトにある必要があります。
具体的には、currentdialogueを介してモンスターのダイアログで使用する場合、 それを設定したスクリプトではなく、関数はモンスタースクリプト内に存在する必要があります。


[speed:x] Makes the text handler print x characters over every 4 frames.
The default text speed in CYF ([speed:1]) is 1 character every 4 frames.

For example, [speed:4] will type text at 4x the regular speed: 4 characters every 4 frames, or 1 per frame.

* Supports fractional numbers! [speed:0.25] writes text at 1/4th the regular speed, for instance.

Overrides [waitall:x].


[letters:x] For this frame only, the text box will show the next x characters.


[name] このテキストは、プレイヤーの名前に置き換えられます。あー...Player.nameが使えることを忘れていましたが、 もうこれでもそれでもどっちでもいいです。


[music:x] Plays the music given as an argument on the main music channel, unless the argument is one of these keywords:

  • play = Only plays the last played music, from the beginning.

  • pause = Pauses the music.

  • unpause = Unpauses the music.

  • stop, null, nil or an empty string = Stops the current music.


[sound:x] Plays the sound given as an argument.


[mugshot:face], [mugshot:null] or [mugshot:{face1,face2,...,time}] Overworld only.

Displays a set face sprite (or sequence of face sprites) next to your dialogue set by General.SetDialog.
Note that the overworld's automatic line break system will not account for face sprites set this way; instead, it's recommended to set a face sprite in General.SetDialog (see The General Object).

This function loads face sprites from YOUR MOD/Sprites/Mugshots. You may put folder names in here, such as [mugshot:Papyrus/suspicious] for YOUR MOD/Sprites/Mugshots/Papyrus/suspicious.png.
Use [mugshot:null] to revert the overworld text box back to normal.


To use a sequence of face sprites, enter something such as
[mugshot:{Papyrus/happy,Papyrus/suspicious,0.1}].
This is a list of normal paths to put into [mugshot:x] as described above, followed by the amount of real time seconds between faces (such as with sprite.SetAnimation).

If you do not provide a time, or the time you enter is <= 0, a default value of 0.2 will be used.


[health:x,y] Heals the Player by x points. Can damage the player if x is negative, but the Player's HP will not go under 1 unless certain conditions are met. The parameter y is optional.

Examples:

  • [health:3] => Heals 3 HP.

  • [health:-5] => Damages 5 HP. Doesn't go under 1HP.

Tags:
  • x (y is unused if these tags are used):
    • kill = Kills the Player.

    • Max-1 = Sets the Player's HP at Player's Max HP minus 1.

    • Max = Sets the Player's HP at Player's Max HP.

  • y:
    • killable = The possible HP reduction can now kill the Player.

    • set = The amount of HP given in the first argument will be the Player's current HP.

Command tags




These tags can be added after a text command and have different behaviours.
To use them, you must do this: [command:tag] or [command:arguments:tag]. For now, only one tag can be added at a time.


skipover Makes a command only be activated when the text is not skipped (by using any [instant] command or the player's skip if playerskipdocommand is set to true). Has no effect on [w] and [letters].


skiponly The command this is attached to will only be activated if the text has been skipped by using any [instant] command, or if the player skips it manually and playerskipdocommand is set to true. This tag disables the [w] and [letters] commands if used on them.
