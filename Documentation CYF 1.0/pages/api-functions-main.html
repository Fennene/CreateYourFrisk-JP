関数 & オブジェクト



このセクションでは、UnitaleがLuaスクリプトに追加した、ゲームを操作する様々な関数を紹介します。 全ての関数には、使用できるスクリプトを示すために、角かっこで囲まれたものがあります。
詳細についてはこのドキュメントの読み方を参照してください。

全てのスクリプトで使用できる関数


DEBUG( text) [E/M/W] デバッグコンソールに、テキストを表示させます。(F9で表示切替) 初めてテキストを表示させた時に、自動的に表示されます。 これを使用して、変数の値を確認したり、コードが実行されているか確認したりできます。


EnableDebugger( bool) [E/M/W] デバッグコンソールの表示を許可または禁止します。 falseを引数に指定すると、デバッグコンソールは閉じられ、表示されなくなります。 私が試したときはうまくいかなかった。 by 翻訳者 trueを指定すると、デバッグコンソールが強制的に開かれることはありませんが、ユーザーがF9を押すことで表示できるようになります。

デバッグコンソールはデフォルトで有効になっています。
注: デバッグコンソールはFPS表示を含みます。デバッグコンソールの有効/無効はFPSの表示にも適応されます。


SetGlobal( your_variable_name, value) [E/M/W] どのスクリプトでもアクセスできる変数を定義します。エンカウンタースクリプトで変数を設定してアクセスする手段とはまた別の手段です。(The Script Objectを参照)

設定した変数はいつでもどこでもGetGlobal(variable_name)で取得できます。


GetGlobal( your_variable_name) returns variable [E/M/W] SetGlobal()で設定した変数を取得します。エンカウンタースクリプトで変数を設定してアクセスする手段とはまた別の手段です。(The Script Objectを参照)


SetRealGlobal( your_variable_name, value) [E/M/W] Sets a Global variable that is accessible even in the overworld. After setting it, you can retrieve it from all of your scripts at any time with GetRealGlobal(variable_name).

Persists through battles, but not between sessions. These variables can be saved using the overworld save system.

* Note: "Complex" variables (such as tables, functions and userdata) can not be saved as Real or AlMighty globals.

* Also note: as Real and AlMighty Globals persist across mods, it is possible for mods to read each others' globals. Be careful when choosing global names.

Can be used in the overworld.


GetRealGlobal( your_variable_name) returns variable [E/M/W] Gets a Global that you previously set using SetRealGlobal().

Can be used in the overworld.


SetAlMightyGlobal( your_variable_name, value) [E/M/W] AlMighty Globals(オールマイティーグローバル)は、設定するとすぐにファイルに保存されるデータです。 CYFを閉じてもデータは残ります。.

しかし、オプション(modDev?)のRemove AlMighty Globalsで削除することができます。

* 注1: 「複雑」な変数/値(配列や関数、ユーザーデータなど)は Real または AlMighty global として保存できません。

* 注2: Real Global と AlMighty Global はCYFがデータを保持するため、他のModに読み込みや書き取りされる可能性があります。 名前を付ける場合は気を付けてください。

Can be used in the overworld.


GetAlMightyGlobal( your_variable_name) returns variable [E/M/W] SetAlMightyGlobal()で設定したデータ(変数)を取得します。
Can be used in the overworld.


SetFrameBasedMovement( bool) [E/M/W] 引数にtrueを指定すると、プレイヤーの動きが時間ベース(120px/s)から、フレームベース(2px/frame)になります。
フレームベースから時間ベースに戻したい時はfalseを引数に指定します。

デフォルトでは、プレイヤーの動きは時間ベースです。(SetFrameBasedMovement(false)が呼び出された時と同じです。)
Note that this function only controls the player's movement with the default control scheme (see Player.SetControlOverride in The Player Object).


SetAction( "FIGHT", "ACT", "ITEM" or "MERCY") [E/M/W] Used alongside State("ENEMYSELECT") (or EnteringState when entering the same state) to force the player to choose FIGHT or ACT. This controls whether you'll see the enemy's health bar in the menu and whether the next state upon pressing Z is ATTACKING or ACTMENU.

ACTIONSELECT時に使用すると、プレイヤーを指定したボタンの上に移動させます。


AllowPlayerDef( bool) [E/M/W] 引数にtrueを指定すると、Undertaleのように、プレイヤーが受けるダメージは５DEFにつき１ダメージ分減少し、LV１の場合は防御力がダメージに影響しません。
受けるダメージは必ず１以上であり、ダメージを完全に防ぐことはできません。
デフォルトではFalse。


SetPPCollision( bool) [E/M/W] 引数にtrueを指定すると、全ての弾丸(発射体(Projectile))は「The Pixel-Perfect collision システム」に基づいて、当たり判定を調節します。 これを使用したい場合は、The Pixel-Perfect Collision Systemを参照することを強くお勧めします。
デフォルトではFalse。


BattleDialogue( list_of_strings) [E/M/W]
or BattleDialog これを使用すると、ダイアログボックスに指定したテキストを表示します。引数に文字列の配列を指定すると、複数回にわたって表示されるようになります。
テキストの表示が終了すると、自動的に状態がモンスターの会話に移ります。 以下の例はベジトイドに実装する例です。

CYF v0.6.4以降、１度に４行以上のテキスを表示する場合、 テキストが上に移動して(１行過剰につき、９ピクセル)、テキストボックス内に収めようとします。

* \nでアスタリスク(*)付きの改行、\rでアスタリスク無しの改行ができます。 

function HandleCustomCommand(command)
    --if command == "DINNER" then
    if command == "ディナー" then
        if ate_greens then -- "ate_greens"はもちろんデフォルトで存在する変数ではないよ
            currentdialogue = {"みどりの\nやさいを\nたべたね"}
        else
            currentdialogue = {"みどりの\nやさいを\nたべてね"}
        end
        BattleDialog({"おなかを　さすった。\nベジトイドが　からだにいい\rしょくじを　だして　くれるらしい。"})
    end
end


State( state_to_go_to) [E/M/W] どんな時でも([要検証])指定された状態へ即座に偏移する強力な関数。
DefenceEnding()でState("ENEMYDIALOGUE")を呼び出したらエラーが出た覚えがある。 by 翻訳者
以下は、引数に指定できる文字列とそれが表す状態、及び状態の意味を示した一覧です。 引数の文字列は、大文字小文字区別しませんが、可読性の点から大文字で渡すことを推奨します。

Arguably the best part about State is that it can be used in conjunction with the text command [func] to change the state from within your dialogue. An example is below - this will have a monster say that he will not fight you, then return to the action select screen, instead of starting a wave. 

currentdialogue = {"I won't fight you.", "[func:State,ACTIONSELECT]"}
  • ACTIONSELECT - プレイヤーのターンを表します。プレイヤーは FIGHT/ACT/ITEM/MERCY を選択できます。

  • ENEMYDIALOGUE - この状態が始まると、アクティブなモンスターのモンスタースクリプトからcurrentdialogueが読み取られ、 敵の会話を表示します。currentdialogueが設定されていない場合は モンスタースクリプトのrandomdialogueからランダムに参照します。

  • DEFENDING - 敵のターンを表します。この状態が始まると、エンカウンタースクリプトのnextwavesが読み取られ、 Wavesフォルダ内の名前が一致するファイルがロードされ、攻撃が始まります。 この時、 wavetimerもエンカウンタースクリプトから読み込まれます。

利用可能ないくつかの状態が存在しますが、時と場合によってはバグなどが発生する恐れがあります。 自己責任で使用してください:

  • ATTACKING - プレイヤーの攻撃時を表します。この状態が始まると、プレイヤーがZ(Enter)キーでモンスターを攻撃するか、 攻撃をミスするかの表示になります。プレイヤーの攻撃対象はENEMYSELECTかACTMENUで最後に選んだ モンスターになります。この状態が終了すると、自動的にENEMYDIALOGUEに偏移します。個人的に１番バグりやすいと思う by 翻訳者

  • ENEMYSELECT - プレイヤーが敵を選択する画面の時を表します。この状態では、戦闘内のアクティブな敵のリストが表示されます。 FIGHTまたはACTを選択すると自動的にこれに偏移します。 プレイヤーがFIGHTを選択した場合、この状態で"Z(Enter)"を押すとATTACKINGに、 ACTを選択した場合は、ACTMENUに移動します。 Stateを使って呼び出した場合、"Z(Enter)"を押しても何も起こりません。

  • ACTMENU - ACTのメニューを表します。この状態では、敵に対する行動コマンドが並べられて表示されます。 敵のcancheckがtrueの場合、自動的にコマンドに"Check"が表示されます。 また、Basic Setupに示されているcommandsの構文に従います。[翻訳力不足] ６つ以上のコマンドが存在するとバグります。 (追記: 選択される敵はENEMYSELECTで最後に選んだ敵になります。)

  • ITEMMENU - この状態では、インベントリのアイテムが表示されています。 これはHandleItemと結びついています。(Game eventsを参照)

  • MERCYMENU - この状態では、プレイヤーが"Spare(逃がす)"または"Flee(逃げる)"を選択できます。 アクティブなモンスターの内１体でもcanspareがtrueになっていると、"Spare"が黄色になります。その状態で"Spare"を選択すると、モンスターを見逃せます。 見逃せない状態で"Spare"を選択すると、エンカウンターの関数(イベント)、HandleSpareが呼び出されます。 また、"Flee"を選択するといくつかのばかげたメッセージが見れますよ(。・ω・。)
    If the encounter was entered from the overworld, choosing "Flee" allows the player to flee the battle and return to the overworld.

  • DIALOGRESULT - これは、BattleDialogが呼び出された時や、 戦闘に勝利した時のメッセージが表示された時、プレイヤーが逃げられなかった時、アイテムを使用した時に偏移される状態です。 全てのテキストが表示され、プレイヤーが"Z(Enter)"を押すと、ENEMYDIALOGUEに偏移します。 この状態の間はプレイヤーのタマシイは強制的に非表示になります。

最後に、エンジン自体を制御する２つの「特別な」状態を紹介します:

  • DONE - 状態をDONEにすると戦闘(ゲーム)を終了します。通常はMod選択画面に戻ります。 If the encounter was entered from the overworld, the battle will end and the player will return to the overworld, in the same way as if the player had ended the battle normally, such as by sparing all the enemies or running away.

  • NONE - 文字通り何もしません。戦闘の最初の１フレームに使用されますが、手動でこれを呼び起こすと、戦闘がフリーズします。Unitale/CYFに何もさせたくない場合に使います。

  • RESETTING - DEPRECATED, DO NOT USE. Only listed for completion purposes. It only exists in Unitale 0.2.1a, so trying to use it in CYF won't work. It was removed in the Github release of Unitale 0.2.1a, and seemed to be only for testing.

Freezing states As of CYF v0.6.2.1, calling State("PAUSE") will perfectly "pause" an encounter. The last active state will remain active, but in a frozen state, until you call State again.

* Only the Update function of the Encounter script will remain active here.

* To unfreeze a state, you must use State to switch to another state, preferably the last active state. GetCurrentState() will tell you the name of the frozen state.


GetCurrentState() returns [E/M/W] 現在の状態を名前を返します。(上記参照)

エンカウンタースクリプトで使用できる関数


RandomEncounterText() returns [E] Select a random monster from the encounter, then get a random entry from the comments table there. You'll want to use this to replicate default encounter behaviour. See code below (or one of the example encounters). 

function DefenseEnding() --This built-in function fires after the defense round ends.
    encountertext = RandomEncounterText()
end


SetButtonLayer( layer) [E] FIGHT/ACT/ITEM/MERCYの各ボタンとプレイヤーの名前、LV、HPを表示するレイヤーを指定の位置に移動させます。
指定可能なレイヤーは、エンジンが指定してあるレイヤーと、定義したスプライトレイヤーです:

  • "Bottom": 何もかも、全てにおいて下

  • "BelowUI": 背景の上

  • "BelowArena": 背景とUIの上

  • "BelowPlayer": 背景、UI、アリーナの上

  • "BelowBullet": 背景、UI、アリーナ、プレイヤーの上

  • "Top": 全てにおいて上

注: "default"を指定すると、各ボタンとUIが元の位置に戻ります。

モンスタースクリプトで使用できる関数


SetSprite( filename) [M] モンスターのスプライトを、Spritesフォルダ内にあるfilename.pngにします。 [func]でこれを使用して、会話の途中でスプライトを変えることができます。


SetActive( active) [M] 引数にfalseを指定すると、モンスターは表示されたままですが、メニューに表示されなくなり、会話しなくなり、イベントを実行しなくなります。 これを使用して、途中でモンスターを戦闘に参加させることができます。 全てのモンスターが非アクティブであると、バグ及びエラーが発生する可能性がとても高いです。


SetDamage(amount) [M] モンスターが次に受けるダメージを設定できます。負の値にできます。
amountに敵が受けるダメージを指定するか0を指定して、攻撃をMISSにできます。[要検証]

この関数は、Game EventのBeforeDamageCalculation内か Player.ChangeTargetを使用する直前にのみ使用できます。
また、SetActive(false)のモンスターに使用しないでください。


Kill() [M] モンスターを即座に倒します。これが最後の敵であれば、戦闘は終了します。
Game EventのOnDeathを呼び出しません。


Spare() [M] モンスターを即座に見逃します。Kill()と同様に、最後の敵であれば戦闘を終了します。
Game EventのOnDeathOnSpareを呼び出しません。


Move( x, y) [M] Moves the enemy's sprite relative to its current position.


MoveTo( x, y) [M] Moves the enemy's sprite relative to the bottom left corner of the screen.
This is effectively the same as setting enemypositions again, except x is 320px left and y is 231px down.


BindToArena( bind, isUnderArena = false) [M] 敵のスプライトがアリーナの動きに従うか設定します。

bindをtrueに指定すると、敵はアリーナの親になり、アリーナと共に動くようになります。
それ以外の場合、isUnderArenaの値によって、敵がアリーナの前か後ろに表示されます。


SetBubbleOffset( x, y) [M] Makes the enemy's dialogue bubble appear x pixels horizontally and y pixels vertically relative to its original position.


SetDamageUIOffset( x, y) [M] Changes the offset of the enemy's damage UI (the enemy's health bar and the red numbers). Note that the damage UI is on a layer above that of the arena.


SetSliceAnimOffset( x, y) [M] Changes the offset of the attack animation (the red slice by default) for when the player attacks this monster.

ウェーブスクリプトで使用できる関数


EndWave() [W] ウェーブを即座に終了します。この関数はUpdate関数(イベント)内のみで使用できます。
